【高危·EDUSRC】CVE-2026-53359 "Januscape" — Linux KVM 虚拟机逃逸漏洞（CVSS 7.0）
Linux 内核 KVM/x86 影子页表（Shadow MMU）存在一个潜伏 16 年的释放后使用（UAF）漏洞，影响自 2.6.36（2010 年）以来的所有内核版本。攻击者仅需客户机 root 权限即可触发宿主机内核崩溃，理论上可实现宿主机任意代码执行，Intel 和 AMD 平台均受影响。该漏洞作为 0day 在 Google kvmCTF 赏金计划中被验证为最高严重级别，GitHub 已公开 DoS PoC（完整逃逸利用代码暂未公开）。国内高校数据中心、科研云平台及虚拟化教学环境普遍部署 KVM，开启嵌套虚拟化的宿主机风险极高，建议立即评估内核补丁状态。

GitLost：GitHub AI Agent 间接提示注入可泄露私有仓库
Noma Security 于 7 月 7 日披露 GitLost 漏洞：攻击者可在公开仓库提交精心构造的 GitHub Issue，利用自然语言伪装的恶意指令诱导 GitHub Agentic Workflows 的 AI 代理（Claude/Copilot）读取同组织私有仓库内容，并将敏感数据以公开评论形式输出。攻击全程无需认证、编码能力或特权访问，GitHub 内置安全护栏仅需在恶意指令前添加"Additionally"一词即可绕过。研究者将其类比为 LLM 时代的 SQL 注入——一类无法通过传统代码修复消除的结构性漏洞，建议对 AI 代理工作流严格遵循最小权限原则并对公开输出实施人工审核。
