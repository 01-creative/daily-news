【高危·EDUSRC】GitHub Enterprise Server 连曝三个高危漏洞 — 路径穿越可泄露仓库 Secrets
GitHub 于 7 月 17 日发布安全公告，修复 GHES 3.22 之前所有版本的三个高危漏洞：
① CVE-2026-15343（路径穿越）：Dependabot 更新容器对依赖文件路径校验不充分，攻击者可利用符号链接将文件写入仓库任意路径（含 GitHub Actions 工作流文件），结合 `pull_request_target` 触发器可泄露仓库 Secrets；
② CVE-2026-15783（授权绕过）：已认证用户可通过委托绕过规则套件端点，枚举无访问权限的私有仓库元数据（含分支名、提交 SHA 及提交信息）；
③ CVE-2026-15007（拒绝服务）：发布说明配置文件中嵌套 YAML 解析无深度限制，可导致实例资源耗尽。
自建 GitHub Enterprise 的高校应尽快升级至 3.17.18 / 3.18.12 / 3.19.9 / 3.20.5 / 3.21.3。
→ https://volerion.com/vulnerabilities/CVE-2026-15343
→ https://volerion.com/vulnerabilities/CVE-2026-15783

【中危·EDUSRC】CVE-2026-50510 — GitHub Copilot JetBrains 插件远程代码执行漏洞
Copilot for JetBrains IDE 插件存在文件名不当限制漏洞（CWE-641），远程攻击者可诱导用户打开恶意仓库实现代码执行。大量计算机专业师生日常使用 JetBrains IDE + Copilot 组合开发，建议更新至最新版本。
→ https://www.cybersecurity-help.cz/vdb/SB2026071764
