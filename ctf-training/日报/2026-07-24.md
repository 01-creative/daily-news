【高危·EDUSRC】Redis Stream NACK 双重释放远程代码执行漏洞（QVD-2026-44936，CVSS 8.8）
该漏洞是此前 CVE-2026-25243 / CVE-2026-25589 的补丁绕过，位于 Redis Stream 消费者组共享 NACK 处理逻辑中。攻击者通过构造恶意 RDB 数据，利用 RESTORE 命令触发双重释放（double-free），最终劫持进程执行任意系统命令。受影响版本为 Redis 6.2.22、7.4.9、8.6.4，修复版本为 7.4.10 / 8.6.5 / 8.8.0。截至发稿未见在野利用，但 PoC 已在 GitHub 公开。Redis 作为高校 Web 应用中广泛使用的缓存与 Session 存储组件，暴露风险较高——尤其是未启用认证或使用弱口令的实例。建议立即升级，并通过 ACL 或 rename-command 禁用 RESTORE、XGROUP 等高危命令。
