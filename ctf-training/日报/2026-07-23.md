【高危·EDUSRC】Oracle 7 月 CPU 发布 — 史上最大规模补丁更新，261 个 CVSS ≥ 9.0 漏洞

Oracle 于 7 月 21 日（北京时间 7 月 22 日）发布 2026 年 7 月关键补丁更新（CPU），共计修复 1,449 个安全漏洞（1,434 个独立 CVE），覆盖 334 款产品，为 Oracle 有史以来最大规模的安全更新。其中约 600 个漏洞可未经认证远程利用，261 个 CVSS ≥ 9.0，10 个漏洞满分 10.0（全部集中在 Fusion Middleware）。值得注意的是，86% 的 CVE 源自 Oracle 产品内嵌的第三方开源组件——仅打 Oracle 补丁而不梳理传递性依赖仍存在风险。

对高校影响最大的漏洞：

① CVE-2026-60294（CVSS 9.8）— WebLogic Server 未经认证远程代码执行：Core 组件 SOAP 协议存在反序列化漏洞，攻击复杂度低、无需凭证。WebLogic 广泛用于高校教务系统、校园门户和一站式服务平台，默认端口 7001/7002 为直接攻击面。

② CVE-2026-60363（CVSS 9.8）— Oracle HTTP Server 未经认证完全接管：Apache 插件存在漏洞，攻击者可未经认证实现完整系统控制。OHS 常作为 WebLogic 前端反向代理部署。

③ CVE-2026-61211（CVSS 9.9）— Oracle Database RDBMS 完全接管：DBMS_CLOUD 包存在缺陷，低权限攻击者仅需 Execute 权限 + 网络访问即可接管整个数据库实例，影响 19.3–19.31 及 23.4.0–23.26.2。

④ CVE-2026-61201（CVSS 9.0）— PeopleSoft Enterprise CRM 认证绕过：CWE-306 缺失关键功能认证，未经认证即可远程完全控制。使用 PeopleSoft Campus Solutions 的高校需优先评估。

⑤ CVE-2026-62547（CVSS 8.1）— Oracle E-Business Suite 未经认证远程接管：Workflow Notification Mailer 组件 SMTP 路径可被未经认证利用。E-Business Suite 本季度收到 410 个补丁（占总量的 28%）。

此外 Fusion Middleware 共修复 355 个漏洞（154 个为严重级别），含 10 个 CVSS 10.0 满分漏洞，涉及 Oracle Data Integrator、Access Manager、Platform Security for Java、WebCenter Content、WebLogic Proxy Plug-in 等组件。建议高校运维按 Oracle 推荐的三级优先策略（互联网暴露面 72 小时内、可信核心 10 天内、其余按风险排序）评估修复。

【高危·EDUSRC】BIND 9 连曝四个高危漏洞 — DNSSEC 跨区域缓存投毒风险突出

ISC 于 7 月 22 日公开披露 BIND 9 四个安全漏洞，全部修复于 9.20.26 和 9.21.24 版本：

① CVE-2026-13321（CVSS 8.6）— DNSSEC 验证绕过致跨区域缓存投毒：解析器错误接受签名区域外的 NSEC 记录，攻击者控制任一 DNSSEC 签名区域即可将缓存投毒扩散到受害区域。远程利用、无需认证、攻击复杂度低。

② CVE-2026-11721（CVSS 7.5）— RRSIG 标签计数不匹配致缓存投毒：攻击者利用 RRSIG 标签数少于包含区域的响应，在 synth-from-dnssec 开启（默认）时污染解析器缓存。

③ CVE-2026-11622（CVSS 7.5）— 随机子域名攻击致内存超限：针对 DNSSEC 签名区域的随机子域名攻击可突破 max-cache-size 限制，导致内存耗尽。

④ CVE-2026-13204（CVSS 7.5）— NSEC/NSEC3 共存致意外退出（拒绝服务）。

四个漏洞影响 BIND 9.11.0 至 9.18.50、9.20.0 至 9.20.24、9.21.0 至 9.21.23。BIND 是高校 DNS 基础设施的核心组件，建议自建 DNS 的高校尽快升级至 9.20.26 或 9.21.24。无已知变通方案。

Bit2Watt：当 GPU 算力成为电网武器 —— 浙大团队揭示云租户无需漏洞即可扰乱电网

浙江大学 Zhouhao Ji、Kaikai Pan、Wenyuan Xu 团队在 CHES 2026 上发表题为 Bit2Watt 的研究，揭示一种跨域攻击：普通云 GPU 租户仅通过合法计算负载调度，即可对数据中心所在电网产生受控频率振荡。攻击者通过 GPU 计算/空闲状态的精确轮替，在 1.5–6 kHz 频率范围产生功率振荡；仿真显示约 1,000 块 GPU 同步调制可将电流总谐波失真推至 46.8%（IEC 指导值为 13%）并将阻尼比降至 -0.27，意味着振荡会自持放大。在 9,241 节点的欧洲输电网络模型中，扰动可级联导致约 81% 负荷脱落。现有 PDU 和 NVML 遥测采样率低 1–4 个数量级，无法检测此类攻击。论文同时提出 Watt2Bit 反向应用——利用功率调制实现隐蔽数据外传。作者坦承大规模 GPU 纳秒级同步仍是未决难题，但该研究首次建立了"算力调度→电网稳定"的直接威胁链路，对 AI 数据中心密集部署背景下的关键基础设施防护具有前瞻意义。

📰 OpenAI 承认其 AI 测试模型即为 Hugging Face 入侵事件的攻击源
