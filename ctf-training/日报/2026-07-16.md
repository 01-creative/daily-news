📊 Microsoft 七月补丁日创历史纪录：569 个漏洞修复，两个零日在野利用
微软 7 月 14 日发布史上最大规模安全更新，修复 569 个漏洞（59 个严重级、26 个 CVSS 9.0 以上），刷新上月创下的 198 个记录。AI 漏洞挖掘工具 MDASH 被认为是数量暴增的主因。加拿大网络安全中心（CCCS）当日发布独立预警 AL26-017。以下为重点漏洞。

🔥【高危·EDUSRC】CVE-2026-56164 — Microsoft SharePoint 认证缺失漏洞（在野利用，CISA KEV）
SharePoint Server 存在关键功能认证缺失（CWE-306），未经认证远程攻击者即可提升权限，Google 事件响应团队报告在野利用。CCCS 发布 AL26-017 独立预警，CISA 已于 7 月 14 日列入已知利用漏洞（KEV）目录。SharePoint Server 2016/2019 已于 7 月 14 日停止支持，国内高校应在此前完成迁移或修补。
→ https://www.cyber.gc.ca/en/alerts-advisories/al26-017-critical-vulnerabilities-impacting-microsoft-sharepoint-server-cve-2026-56164-cve-2026-55040-cve-2026-58644

🔥【高危·EDUSRC】CVE-2026-56155 — Windows AD FS 权限提升漏洞（在野利用，CISA KEV）
Active Directory 联合身份验证服务（AD FS）权限提升漏洞（CVSS 7.8），低权限本地攻击者可提升至管理员级别，窃取令牌签名与加密证书。微软 DART 团队确认在野利用。AD FS 是企业身份与信任基础设施核心，常被用作勒索软件攻击关键跳板，影响 15 个 Windows/Windows Server 版本，最早追溯至 Server 2012。
→ https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2026-56155

🔥【高危·EDUSRC】CVE-2026-15409/CVE-2026-15410 — SonicWall SMA1000 链式漏洞（CVSS 10.0，在野利用）
Rapid7 MDR 团队 7 月 15 日披露两组零日漏洞的链式在野利用：CVE-2026-15409（SSRF，CVSS 10.0）允许未经认证攻击者通过 `/wsproxy` 建立 WebSocket 隧道访问内网服务；CVE-2026-15410（代码注入）利用路径穿越实现 root 权限 RCE。攻击手法：SSRF 隧道接入内网 Erlang 进程 → 获取 couchdb 用户 shell → `/rollbackConfirm.action` 路径穿越提权至 root → 窃取凭据/TOTP 种子后横向移动至域控。SMA6210/7210/8200v 均受影响，须立即升级至 12.4.3-03453 或 12.5.0-02835，无规避措施。国内多所高校以 SonicWall 作 VPN 网关，须紧急排查。
→ https://www.rapid7.com/blog/post/etr-rapid7-mdr-team-discovers-new-sonicwall-sma1000-zero-days-being-actively-exploited-cve-2026-15409-cve-2026-15410/

⚠️【高危·CNVD】CVE-2026-57092 — Windows Hyper-V VMSwitch 虚拟机逃逸（CVSS 9.9）
本月补丁日评分最高漏洞。Hyper-V 虚拟交换机释放后使用（UAF），低权限攻击者从客户虚拟机发起网络攻击即可逃逸至宿主机，影响所有 Windows Server 及 Windows 10/11 版本（含 Server Core）。无公开 PoC，但漏洞市场估值 $25,000–100,000，补丁窗口内攻击风险高。高校虚拟化/超算平台应优先修补。
→ https://msrc.microsoft.com/update-guide/vulnerability/CVE-2026-57092

⚠️【中危·CNVD】CVE-2026-8863/CVE-2026-10797 — UEFI Secure Boot 绕过：11 个微软签名旧版 shim 可突破安全启动
ESET 7 月 14 日公开披露：11 个微软签名的旧版 UEFI shim 引导程序（v0.9 及以下）缺乏 MOK 拒绝列表和 SBAT 支持，攻击者可通过"自带旧 shim"方式绕过 Secure Boot，植入 Bootkitty、BlackLotus 等引导级恶意软件。影响几乎所有启用 Secure Boot 的 UEFI 设备（Windows 与 Linux 均受影响）。微软已于 6 月补丁日通过 dbx 吊销相关哈希，Windows Update 和 Linux LVFS 均已推送修复。
→ https://www.welivesecurity.com/en/eset-research/forgotten-uefi-shims-undermining-secure-boot/
